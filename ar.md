# X-Craft — منصة إدارة تويتر/X المدعومة بالذكاء الاصطناعي

> نظام ذكاء اصطناعي متكامل يعمل محلياً على جهازك لإدارة حسابك على X (تويتر) — إعادة كتابة التغريدات، توليد المحتوى، الجدولة، والتحليل العميق. مبني بـ Next.js 15، LM Studio، وتصميم ذهبي على أسود مع خط أميري.

## ١. رؤية المشروع

X-Craft بُني حول فكرة محورية واحدة: **مساعدك بالذكاء الاصطناعي يجب أن يعمل على جهازك أنت، يعالج بياناتك محلياً، بدون أي تعرّض لخوادم خارجية.**

كل عمليات الذكاء الاصطناعي تمر عبر **LM Studio** — خادم LLM محلي متوافق مع OpenAI API. تغريداتك ومسوداتك وتحليلاتك لا تغادر جهازك أبداً لتصل لأي مزود ذكاء اصطناعي سحابي.

المنصة مبنية في المقام الأول لـ **المستخدمين الناطقين بالعربية** (اللهجة السعودية بشكل خاص) مع دعم كامل لإنشاء المحتوى باللغة الإنجليزية. اتجاه الواجهة يمين لليسار (`dir="rtl"`) وكل مكوّن مُختبر في سياقات النصوص العربية واللاتينية.

## ٢. قائمة الميزات الكاملة

### الميزات الأساسية (كما طلبها المستخدم)

| # | الميزة | الوصف |
|---|--------|-------|
| ١ | **تسجيل الدخول بتويتر OAuth** | تدفق OAuth 2.0 PKCE الكامل عبر NextAuth v5، الصلاحيات: قراءة، كتابة، متابعون |
| ٢ | **إعادة كتابة التغريدة** | الصق أي رابط تغريدة → الذكاء الاصطناعي يجلبها ويحلل لغتها ويعيد كتابتها |
| ٣ | **ترجمة للهجة السعودية** | تغريدة إنجليزية → عربية بتعابير سعودية أصيلة (خوي، والله، بس...) |
| ٤ | **ترجمة عربي → إنجليزي** | تغريدة عربية → إنجليزية احترافية نظيفة |
| ٥ | **تعديل وتحسين** | الحفاظ على المعنى الأصلي مع تحسين الأسلوب والتأثير |
| ٦ | **دعم الخيوط (Threads)** | اكتشاف روابط الخيوط، إعادة كتابة كل الأجزاء مع الحفاظ على التسلسل المنطقي |
| ٧ | **حفظ الوسائط** | وصف روابط الوسائط من التغريدة الأصلية للذكاء الاصطناعي لكتابة سياقية |
| ٨ | **توليد تغريدة بالذكاء الاصطناعي** | اكتب فكرة عن موضوع → AI يولّد 3 نسخ من التغريدة عالية التفاعل |
| ٩ | **تقويم المحتوى** | عرض شهري/أسبوعي بتصميم Google Calendar لتغريداتك المجدولة |
| ١٠ | **أفضل وقت للنشر** | فترات زمنية مبنية على الخوارزمية وأنماط نشاط الجمهور السعودي |
| ١١ | **جدولة التغريدات** | جدولة تغريدات مفردة أو خيوط كاملة لأيام وأسابيع وأشهر |
| ١٢ | **لوحة التحليلات** | مخططات: نمو المتابعين، المشاهدات، الإعجابات، الريتويت، معدل التفاعل |
| ١٣ | **محلل الهاشتاقات** | نقاط الترند، مستوى المنافسة، متوسط الوصول لكل هاشتاق |
| ١٤ | **بناء الخيوط** | منشئ خيوط مرئي بالسحب والإفلات مع معاينة مباشرة |

### الميزات الإضافية (١٥ ميزة أضافها المشروع)

| # | الميزة | الوصف |
|---|--------|-------|
| ١٥ | **مقياس قوة الـ Hook** | نقاط 0–100 في الوقت الفعلي تقيّم قدرة الجملة الأولى على إيقاف التمرير |
| ١٦ | **توقع التفاعل** | AI يتوقع الإعجابات والريتويت واحتمالية الانتشار قبل النشر |
| ١٧ | **مؤشر الانتشار الفيروسي** | نسبة % احتمال انتشار التغريدة خارج المتابعين الحاليين |
| ١٨ | **مكتبة القوالب** | قوالب مدمجة ومحفوظة حسب الفئة (خيط، رأي، نصيحة، قصة) |
| ١٩ | **مدير المسودات** | حفظ وتنظيم والبحث في مسودات التغريدات مع نظام تصنيف |
| ٢٠ | **متابعة المنافسين** | مراقبة حتى 10 حسابات منافسة: المتابعون، وتيرة النشر، أبرز الهاشتاقات |
| ٢١ | **رؤى الجمهور** | توزيع العمر، الجنس، أهم الدول، أوقات الذروة — مرئية بمخططات |
| ٢٢ | **خريطة حرارة أسبوعية** | شبكة 7×24 تعرض درجة التفاعل لكل ساعة في كل يوم من الأسبوع |
| ٢٣ | **محسّن البايو** | AI يعيد كتابة بايو تويتر مع الكلمات المفتاحية والـ CTA ومحاذاة المجال |
| ٢٤ | **خلاصة الترندات** | أحدث الترندات للسعودية مع تحليل المشاعر (إيجابي/محايد/سلبي) |
| ٢٥ | **تقارير الأداء** | تقارير أسبوعية/شهرية مولّدة تلقائياً بأبرز المقاييس |
| ٢٦ | **أداة حالة LM Studio** | مؤشر حي لاتصال الذكاء الاصطناعي المحلي، النموذج المحمّل، زمن الاستجابة |
| ٢٧ | **تبديل متعدد اللغات** | تبديل المحتوى المُولَّد بين اللهجات العربية واللغة الإنجليزية |
| ٢٨ | **محدد المجال** | حدّد مجال حسابك (تقنية، أعمال، رياضة...) لتخصيص كل مخرجات AI |
| ٢٩ | **مركز الإشعارات** | تنبيهات داخل التطبيق: تغريدة نُشرت، وصلت لمئة متابع، تذكير بأفضل وقت |

## ٣. التقنيات والمكتبات المستخدمة

### الإطار والبيئة

| الحزمة | الإصدار | الدور |
|--------|---------|-------|
| `next` | 15.0.3 | إطار React كامل المكدس (App Router) |
| `react` | 18.3.1 | تصيير الواجهة |
| `typescript` | 5.6.3 | أمان الأنواع في كامل قاعدة الكود |
| `node` | ≥ 20 | بيئة التشغيل |

### المصادقة

| الحزمة | الإصدار | الدور |
|--------|---------|-------|
| `next-auth` | 5.0.0-beta.25 | إدارة جلسات OAuth 2.0 |
| `@auth/prisma-adapter` | 2.7.2 | يخزّن الجلسات والحسابات في قاعدة البيانات |

يستخدم NextAuth v5 نمط **App Router handlers** الجديد — ملف `auth.ts` يُصدّر `handlers`, `auth`, `signIn`, `signOut`.

### قاعدة البيانات

| الحزمة | الإصدار | الدور |
|--------|---------|-------|
| `prisma` | 5.22.0 | تعريف المخطط، الترحيل، الاستعلامات الآمنة |
| `@prisma/client` | 5.22.0 | عميل الاستعلام المولَّد |
| SQLite (افتراضي) | — | قاعدة بيانات ملفية محلية (`dev.db`) |

SQLite للتطوير المحلي بدون إعداد. يمكن التبديل لـ PostgreSQL أو MySQL بتغيير `provider` في `schema.prisma`.

### الذكاء الاصطناعي / النماذج اللغوية

| الحزمة | الإصدار | الدور |
|--------|---------|-------|
| `openai` | 4.67.3 | SDK متوافق مع OpenAI — يشير لـ LM Studio المحلي |

LM Studio يعرض REST API متوافقاً مع OpenAI على `http://localhost:1234/v1`. يُهيَّأ SDK بـ `baseURL` يشير لهذا العنوان و `apiKey: "lm-studio"` (يتجاهله LM Studio لكن SDK يتطلبه).

### واجهة برمجة X (تويتر)

| الحزمة | الإصدار | الدور |

| `twitter-api-v2` | 1.18.0 | عميل Twitter API v2 — جلب التغريدات، النشر، الخيوط، الجدول الزمني |

يستخدم التطبيق **OAuth 2.0 PKCE** (وليس 1.0a). الصلاحيات المطلوبة: `tweet.read tweet.write users.read offline.access follows.read`

POST   /api/ai/rewrite                 ← إعادة كتابة تغريدة عبر LM Studio

GET    /api/ai/analyze                 ← فحص حالة اتصال LM Studio
GET    /api/ai/best-time               ← أفضل أوقات النشر للمستخدم

POST   /api/twitter/fetch-tweet        ← جلب بيانات التغريدة من الرابط
GET    /api/twitter/schedule           ← قائمة التغريدات المجدولة (مع فلاتر)
POST   /api/twitter/schedule           ← إنشاء تغريدة مجدولة جديدة
PUT    /api/twitter/schedule           ← تحديث تغريدة مجدولة
DELETE /api/twitter/schedule?id=...    ← إلغاء/حذف تغريدة مجدولة

GET    /api/twitter/analytics          ← بيانات التحليلات + الملخص
GET    /api/twitter/drafts             ← قائمة جميع المسودات
POST   /api/twitter/drafts             ← إنشاء مسودة

POST   /api/twitter/templates          ← إنشاء قالب

## ٧. خوارزميات الذكاء الاصطناعي

جميع استدعاءات AI موجَّهة عبر `src/lib/lmstudio.ts`. الملف يعرّض خمس وظائف رئيسية:

### ٧.١ `rewriteTweet()` — إعادة كتابة التغريدة

**المدخلات:**
- `originalTweet` — النص الخام
- `originalLanguage` — تُكتشف بالـ regex (`ar` | `en` | `other`)
- `mode` — أحد 5 أوضاع إعادة الكتابة
- `tone`، `dialect`، `targetLength`، `audience`، `addHashtags`، `addEmojis`

**الخوارزمية:**

```
١. بناء system prompt:
   - في حالة المخرج العربي:
       • دليل اللهجة السعودية الأصيلة مع تعابير محددة
       • قواعد تفاعل تويتر بالعربية
       • تعليمات JSON فقط في المخرج
   - في حالة المخرج الإنجليزي:
       • نظرية الـ Hook، ميكانيكيات الانتشار، قواعد الـ CTA
       • تعليمات JSON فقط في المخرج

٢. بناء user prompt:
   - تعليمة خاصة بالوضع في اللغة المطابقة
   - نص التغريدة + وصف الوسائط الاختياري
   - ملاحظة الخيط إذا كانت isThread=true
   - مخطط JSON للمخرج المتوقع

٣. استدعاء LM Studio:
   temperature: 0.8 (إبداعي لكن منسجم)
   max_tokens: 2000

٤. تحليل استجابة JSON:
   - الاستخراج عبر regex: /\{[\s\S]*\}/
   - الحقول: rewritten, threadParts, hashtags, explanation, hookScore, engagementTips

٥. الاحتياط:
   - إذا فشل تحليل JSON → إعادة المحتوى الخام كـ rewritten
   - إذا فشل الاتصال → إعادة خطأ منظم برسالة عربية
```

**دليل اللهجة السعودية المُحقَن في system prompt:**
```
استخدم اللهجة السعودية الخليجية الأصيلة مع تعابير محلية مثل:
خوي، والله، عشاني، بس، كذا، شي، ماشي، عساك بخير
```

### ٧.٢ `generateTweet()` — توليد التغريدات

**المدخلات:** `topic`، `language`، `tone`، `format` (single|thread)، `audience`، `keywords`، `niche`، `addHashtags`، `addEmojis`، `threadLength`، `dialect`

**الخوارزمية:**

```
١. system prompt ثنائي اللغة (ar أو en)
٢. طلب 3 نسخ من التغريدة بدقة:
   - النسخة ١: بيان مباشر / رأي
   - النسخة ٢: زاوية القصة / السرد
   - النسخة ٣: سؤال / hook التفاعل
٣. كل نسخة تشمل:
   - content — النص
   - threadParts — إذا كان format=thread
   - hashtags — الهاشتاقات
   - hookScore — 0-100
   - engagementEstimate — التفاعل المتوقع
٤. قسم التحليل:
   - bestPostingTime — فترتان زمنيتان
   - suggestedHashtags — 5 هاشتاقات
   - niche — تصنيف المجال
٥. temperature: 0.9 (إبداعية أعلى)
   max_tokens: 3000
```

### ٧.٣ `analyzeTweetPerformance()` — تحليل الأداء

**المدخلات:** `tweet` (النص الخام)

**الخوارزمية:**

```
١. اكتشاف اللغة بالـ regex
٢. بناء prompt التحليل طالباً JSON:
   {
     hookScore: 0–100,          قوة الجملة الافتتاحية
     readabilityScore: 0–100,   سهولة القراءة
     engagementPrediction: 0–100, توقع التفاعل
     viralProbability: 0–100,   احتمالية الانتشار
     strengths: string[],       نقاط القوة
     weaknesses: string[],      نقاط الضعف
     improvedVersion: string,   نسخة محسّنة
     bestPostingTimes: string[]  أفضل الأوقات
   }
٣. temperature: 0.5 (تحليلي، متسق في النقاط)
   max_tokens: 1500
```

### ٧.٤ `optimizeBio()` — تحسين البايو

**المدخلات:** `bio`، `niche`، `language`

**قواعد تحسين البايو المُحقَنة في الـ prompt:**
```
١. ابدأ بالقيمة التي تقدمها للمتابعين
٢. أضف 3–5 كلمات مفتاحية قابلة للبحث في مجالك
٣. أضف عنصراً شخصياً (شخصية، موقع، إيموجي)
٤. اختم بـ CTA واضح (رابط، DM، اشترك)
٥. ابقَ تحت 160 حرفاً
```

### ٧.٥ `detectLanguage()` — اكتشاف اللغة

اكتشاف سريع بالـ regex (بدون استدعاء AI):
```ts
// اكتشاف الأحرف العربية (Unicode Range)
const arabicPattern = /[\u0600-\u06FF\u0750-\u077F\u08A0-\u08FF]/;
if (arabicPattern.test(text)) return "ar";
// اكتشاف اللاتينية
const englishPattern = /[a-zA-Z]/;
if (englishPattern.test(text)) return "en";
return "other";
```

## ٨. محرك خوارزمية تويتر

في `src/lib/algorithms.ts`. هذه الوحدة تُرمّز سلوك خوارزمية X/Twitter المعروف إلى بيانات قابلة للتنفيذ.

### ٨.١ أفضل أوقات النشر — المملكة العربية السعودية

مصفوفة `SAUDI_BEST_TIMES` تحتوي **14 فترة زمنية مُنقَّطة** تغطي جميع أيام الأسبوع السبعة. النقاط مبنية على:

- روتين الحياة اليومي السعودي (ساعات العمل: الأحد–الخميس، إجازة: الجمعة–السبت)
- أنماط التفاعل التاريخية للمحتوى العربي
- ذروة استخدام الهاتف المحمول (أوقات التنقل، التصفح بعد العشاء)

**أعلى الفترات نقاطاً:**

| النقاط | اليوم | الساعة | السبب |
|--------|-------|--------|-------|
| 95 | الخميس | 22:00 | ليلة الإجازة — أعلى تفاعل عربي في الأسبوع |
| 93 | الجمعة | 21:00 | ذروة التصفح ليلة الجمعة |
| 92 | الثلاثاء | 21:00 | نافذة التفاعل المرتفعة منتصف الأسبوع |
| 90 | الأحد | 21:00 | ارتفاع مسائي بداية الأسبوع |

### ٨.٢ خريطة الحرارة الأسبوعية

`getWeeklyHeatmap()` تولّد شبكة 7×24 (168 خلية) حيث لكل خلية نقاط من 0–100. فترات أفضل الأوقات المعروفة تملأ نقاطها الدقيقة؛ باقي الخلايا تحصل على نقاط خط أساس واقعية مع عشوائية طفيفة لتجنب خرائط متطابقة.

تُعرض خريطة الحرارة كجدول ذي تدرج لوني: خلايا داكنة = تفاعل منخفض، خلايا ذهبية مضيئة = ذروة التفاعل.

### ٨.٣ حاسبة نقاط التغريدة

`calculateTweetScore()` تأخذ خصائص التغريدة وتعيد نقاطاً مركّبة 0–100:

```
النقاط الأساسية: 50

+ 15   طول 20–140 حرف (النقطة المثالية)
+ 10   طول 140–280 حرف
- 10   طول < 20 حرف (قصير جداً)
+ 20   يحتوي وسائط (صورة/فيديو)
+ 15   خيط (Thread)
+ 5/8/6   هاشتاق واحد / اثنان / ثلاثة
- 5    أكثر من 5 هاشتاقات
+ 10   ينتهي بسؤال
+ 5    يحتوي إيموجي
+ 10   محتوى عربي (مكافأة للاستهداف السعودي)

مقيّد: حد أدنى 0، حد أقصى 100
```

## ١٤. نظام الجدولة

### الجدولة اليدوية

يختار المستخدم التاريخ/الوقت من التقويم أو صفحة المجدوِل. مسار `POST /api/twitter/schedule` يتحقق من:
1. `scheduledAt` في المستقبل
2. المحتوى ليس فارغاً
3. عدد الأحرف صحيح (≤ 280 لكل تغريدة، ≤ 25 جزءاً للخيوط)

### النشر التلقائي (Cron)

`GET /api/cron` يُستدعى بخدمة cron خارجية (مثل Vercel Cron أو cron-job.org) كل دقيقة:

```
١. استعلام ScheduledTweet WHERE status='scheduled' AND scheduledAt <= NOW()
٢. لكل تغريدة مستحقة:
   أ. إذا كانت خيطاً: استدعاء postThread(userId, JSON.parse(threadParts))
   ب. غير ذلك: استدعاء postTweet(userId, content)
   ج. عند النجاح: تحديث status='posted'، تعيين tweetId، postedAt=الآن
   د. عند الفشل: تحديث status='failed'، تسجيل الخطأ
٣. إعادة { processed: N, failed: M }
```

المسار محمي بالتحقق من رأس `Authorization: Bearer {CRON_SECRET}`.

### مولّد الجدول الأمثل

عندما ينقر المستخدم "جدولة تلقائية" في صفحة المجدوِل:
1. تحميل جميع المسودات بـ `status=draft`
2. استدعاء `generateWeeklySchedule(tweetsPerDay)` من `algorithms.ts`
3. ربط المسودات بأعلى الفترات نقاطاً للـ 7 أيام القادمة
4. إنشاء سجلات `ScheduledTweet` بالجملة

1. اذهب لـ [developer.twitter.com](https://developer.twitter.com)
2. أنشئ مشروعاً وتطبيقاً
3. فعّل **OAuth 2.0** مع PKCE
4. عيّن رابط الـ callback لـ: `http://localhost:3000/api/auth/callback/twitter`
5. عيّن رابط الموقع لـ: `http://localhost:3000`
6. اطلب الصلاحيات: `tweet.read tweet.write users.read offline.access follows.read`
7. انسخ الـ **Client ID** والـ **Client Secret**

```bash
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000).

```bash
curl -H "Authorization: Bearer YOUR_CRON_SECRET" http://localhost:3000/api/cron
```

للإنتاج، اضبط خدمة الـ cron لاستدعاء هذا المسار كل دقيقة.
